# 기능 구현 노트: 클립보드 붙여넣기 및 Agent Ingest 위임 기능

## 1. 개요
사용자가 클립보드의 텍스트를 웹 UI에 붙여넣으면, 이를 옵시디언 볼트의 `raw/incoming` 경로에 새로운 노트로 저장하고, 동시에 Hermes(또는 등록된 AI 에이전트)에게 `ingest` 명령을 내려 볼트에 정식 등록되도록 위임하는 기능을 구현합니다.

## 2. 요구 사항
- **프론트엔드**: 클립보드 내용을 붙여넣을 수 있는 UI 제공. (제목 옵션, 내용 필수)
- **백엔드**: 텍스트를 받아 `raw/incoming` 하위에 `.md` 파일로 저장하고, 즉시 Hermes API로 위임하는 단일 엔드포인트 구축.

---

## 3. 백엔드 (FastAPI) 수정 계획

백엔드에서는 파일 저장과 에이전트 위임이라는 2단계를 하나의 트랜잭션으로 처리하는 신규 엔드포인트 `/api/ingest/raw`를 추가합니다.

### 3.1. `app/models/note.py` 수정
새로운 요청 모델 추가:
```python
from pydantic import BaseModel
from typing import Optional

class RawIngestRequest(BaseModel):
    content: str
    title: Optional[str] = None  # 제목이 없으면 타임스탬프 기반 자동 생성
```

### 3.2. `app/routes/ingest.py` 수정
기존 라우터에 엔드포인트를 추가합니다. 기존의 `VaultWriter` 인스턴스와 `delegate_ingest_to_hermes` 함수를 조합하여 구현합니다.
```python
import time
from fastapi import APIRouter, Depends, HTTPException
from app.models.note import RawIngestRequest
from app.auth import require_auth
from app.services.hermes_ingest_service import delegate_ingest_to_hermes

# ... 기존 코드 ...

@router.post("/ingest/raw")
async def ingest_raw_clipboard(req: RawIngestRequest, _: str = Depends(require_auth)):
    # 1. 파일명 및 경로 생성
    filename = req.title.strip() if req.title else f"clip_{int(time.time())}"
    target_path = f"raw/incoming/{filename}"
    
    # 2. Vault에 파일 저장
    try:
        status = await writer.ingest(
            target_path=target_path,
            content=req.content,
            tags=["clipboard", "raw"],
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"저장 실패: {e}")

    # 3. Hermes 에이전트에 Ingest 위임
    try:
        hermes_result = await delegate_ingest_to_hermes(target_path)
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"에이전트 위임 실패: {e}")

    return {
        "status": status,
        "path": target_path,
        "hermes_result": hermes_result
    }
```

## 4. 프론트엔드 (Next.js) 수정 계획

클라이언트 컴포넌트를 통해 사용자가 클립보드 내용을 쉽게 붙여넣고 API를 호출할 수 있도록 UI를 구성합니다.

### 4.1. API 클라이언트 (`src/lib/api.ts`)
새로운 엔드포인트를 호출하는 함수 추가:
```typescript
export async function ingestRawClipboard(content: string, title?: string) {
  const res = await fetch(`${API_BASE}/api/ingest/raw`, {
    method: "POST",
    headers: getHeaders(),
    body: JSON.stringify({ content, title }),
  });
  if (!res.ok) {
    const err = await res.json();
    throw new Error(err.detail || "클립보드 저장 및 위임 실패");
  }
  return res.json();
}
```

### 4.2. UI 컴포넌트 신규 작성 (`src/components/QuickPasteModal.tsx`)
클립보드 내용을 입력받는 모달창 또는 패널 구현:
- **Title (Optional)**: 노트 제목 입력란. (input)
- **Content (Required)**: 다중 줄 입력을 위한 `textarea`.
- **Submit Button**: 클릭 시 `ingestRawClipboard` 호출 후 로딩 상태(Spinner) 표시, 완료 후 성공 메시지 표시 및 모달 닫기.

### 4.3. 메인 화면 (`src/app/page.tsx`) 연동
- 상단 헤더의 버튼 그룹이나 검색창(`PromptInput`) 하단에 **"클립보드 수집"** 버튼(예: `ClipboardPaste` 아이콘)을 배치.
- 버튼 클릭 시 `QuickPasteModal`이 팝업되도록 상태(`showPasteModal`)를 추가하여 연동.

## 5. 작업 순서 가이드 (에이전트용)

이 노트를 읽고 작업을 시작하는 에이전트는 다음 순서대로 실행하세요.

1. **모델 추가**: `backend/app/models/note.py`에 `RawIngestRequest` 모델을 추가한다.
2. **라우터 수정**: `backend/app/routes/ingest.py`에 `POST /api/ingest/raw` 라우트 블록을 추가한다.
3. **프론트엔드 API 연결**: `frontend/src/lib/api.ts`에 `ingestRawClipboard` 함수를 추가한다.
4. **UI 컴포넌트 제작**: `frontend/src/components/QuickPasteModal.tsx`를 생성하고 UI 폼과 저장 동작을 연결한다.
5. **UI 통합**: `frontend/src/app/page.tsx`에 붙여넣기 버튼을 달아 모달과 연결한다.
6. **동작 테스트**: `npm run dev`와 `uvicorn`이 켜진 상태에서 프론트엔드 버튼을 눌러 실제 `raw/incoming/...`에 마크다운 파일이 생성되고 Hermes 에이전트 위임 응답이 반환되는지 확인한다.
